# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|:---------:|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | **Абстрактный класс** | Есть общие **поля состояния** (имя, возраст) и отношение «является» (Собака — это Животное). |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | **Интерфейс** | Это способность («умеет» быть сериализованным). Классы совершенно разные, между ними нет иерархии. |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | **Абстрактный класс** | Есть общие **данные** (скорость, уровень топлива), которые логично хранить в родительском классе. |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | **Интерфейс** | Это **поведение** («умеет» подключаться), которое нужно добавить к объектам из разных иерархий. |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | **Абстрактный класс** | Общая структура и наличие состояния (цвет), отношение «является» (Круг — это Фигура). |
| Класс должен одновременно уметь летать, плавать и ездить | **Интерфейс** | В Java невозможно наследовать несколько классов одновременно. Комбинировать разные «навыки» можно только через **множественную реализацию интерфейсов**. |
